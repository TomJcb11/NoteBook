What is Immutable in Tuples?

Unlike Python lists, tuples are immutable. Some Characteristics of Tuples in Python. Like Lists, tuples are ordered and we can access their elements using their index values.

But:

We cannot update items to a tuple once it is created. Tuples cannot be appended or extended.We cannot remove items from a tuple once it is created.

So basicly what appends when you try to add more element inside of a tuple

tuple1= (1,2,3)
tuple2= (4,5,6)
print(f'the tuple1 is located at {id(tuple1)} and has value of {tuple1})')
tuple1+=tuple2
print(f'the tuple1 is located at {id(tuple1)} and has value of {tuple1})')
tuple1+=(7,8,9)
print(f'the tuple1 is located at {id(tuple1)} and has value of {tuple1})')

we can see that each time we update tuple1 ,because a tuple is immutable you cannot directly update its value, python create a new tuple and reaffect the variable tuple1 on it. but each time the id change

the tuple1 is located at 132873796386112 and has value of (1,2,3)
the tuple1 is located at 132873781472192 and has value of (1,2,3,4,5,6)
the tuple1 is located at 132873781493440 and has value of (1,2,3,4,5,6,7,8,9)
the tuple1 is located at 132873782588608 and has value of (1,2,3,4,5,6,7,8,9,1,2,3,4,5,6,7,8,9)

Updating Tuple Values

As explained above, we can't exactly update value of a tuple, in fact we recreate another objet of type tuple and the variable is now pointing to the new object. and the old one (no more used) go to the garbage collector.

Unpack python tuple *args

Static Positionnal only

fruits = ('apple','banana','cherry')
(x, y, z) = fruits
print(y) #type str

Dynamic Positionnal (x,*args)

fruits = ('apple','banana','cherry')
(x,*y) = fruits
print(y) #type list

Another Dynamic Positionnal only (x,*args,y)

fruits = ('apple','banana','cherry','mango')
(x,*y, z) = fruits
print(y) #type list
