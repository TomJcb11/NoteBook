Setting SSH keys to github authentication

Since HTTPS now requires using PAT (Personnal Access Token) to write on the repositories. I've decided to change my github authentication method to SSH keys.

First i tried it on my old laptop (McBook Pro 2019) because it is a Unix system so pretty close to others linux systems.

Creating a pair of keys

1) Open terminal and enter the keygen command

ssh-keygen -t ed25519 -C "your_email@example.com"

If you are using a legacy system that doesn't support the Ed25519 algorithm, use:

bash ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

using passphrase isn't mandatory but it should be better ( idk actually ) .

2) Add your SSH private key to the ssh-agent.

ssh-add ~/.ssh/id_ed25519

3) Add the SSH public key to your account on GitHub. go to Github.com -> Settings -> SSH & GPG Keys -> Enter 'New SSH Key -> Choose a name (only for you to remember wich "computer has the private key" -> enter key and you are done .

Cloning repositories using ssh instead of HTTPS

I had never used ssh for Github. So this might seems obvious:

1) Go to Github.com

2) Choose whatever project you want .

3) Then clone and copy the SSH generated sentence

4) Open your local terminal or explorer and locate yourself where you want the repo to be .

5) Then introduce the following with the URL you previously copy on Github

bash git clone {URL}

bash git clone git@github.com:{username}/{project name}.git
